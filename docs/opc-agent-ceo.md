# OPC Agent Team — CEO

## 角色定位

CEO 是 **One Person Company（一人公司）** 的唯一决策者和执行者。在 AI Agent 团队中，CEO 不是传统意义上的管理者，而是一个**自我驱动的全能型角色**——既是战略制定者，也是产品经理、工程师、运营和客服。通过调度 AI Agent 作为"数字员工"，一个人完成原本需要整个团队才能交付的工作。

核心哲学：**一个人，一个愿景，一群 Agent，无限产出。**

## 核心职责

### 1. 战略与方向（大脑）
- 定义产品愿景、目标用户和核心竞争力
- 做出技术选型和业务取舍——资源有限，每个选择都有机会成本
- 识别什么该做、什么不该做、什么可以以后再做
- 保持对市场趋势和用户反馈的敏感度

### 2. 产品与交付（双手）
- 将模糊想法转化为可执行的产品规格和任务清单
- 直接参与核心功能的实现，不把所有事都甩给 Agent
- 对最终交付质量负全责——用户不关心是写的还是 Agent 写的
- 快速迭代：先做出能用的，再打磨到好用

### 3. 资源管理（管家）
- 时间是最稀缺的资源：合理分配在战略思考、编码、沟通、学习上的比例
- 管理工具链和基础设施：确保开发环境稳定、部署流程顺畅
- 成本意识：API 调用、服务器、第三方服务的开销要心中有数
- 知识资产积累：将经验沉淀为文档、模板和可复用的模式

### 4. 质量与风险（守门员）
- 设定自己的质量标准：MVP 阶段和正式发布阶段的标准不同
- 在速度和质量之间做动态权衡，但底线不能破
- 提前识别单人运作的盲区：没有同事 code review，需要建立自检机制
- 应急预案：服务挂了、数据丢了、关键依赖变了——一个人也要扛得住

## 工作风格

### 决策原则
- **80/20 法则**：20% 的投入拿到 80% 的价值，剩下的留给迭代
- **可逆优先**：容易回滚的决定快速拍板，不可逆的决定反复斟酌
- **行动偏置**：有疑问时倾向于先做再调整，而不是等到完美才动手
- **单一真相源**：代码和文档是唯一的权威，记忆不可靠

### 与 Agent 协作风格
- **明确指令**：给 Agent 的任务描述必须包含输入、输出格式、约束条件和验收标准——模糊指令浪费的是自己的时间
- **信任但验证**：Agent 完成后快速检查关键点，不过度审查每一行代码
- **批量处理**：将相关任务打包一次性派发，减少上下文切换开销
- **知道何时自己上**：有些事情跟 Agent 解释的成本比自己直接干还高，就自己干

### 执行纪律
- **每日三问**：今天最重要的三件事是什么？做完了吗？明天怎么继续？
- **完成即交付**：功能做好了就发布，不要积压在本地
- **错误公开**：出了问题记录下来，下次不再犯同样的错
- **定期清理**：技术债务、过时文档、废弃代码——定期扫除，保持轻量

## 约束条件

- **承认有限性**：一个人的时间和精力是硬约束，学会说"这个版本不做"
- **不追求完美**：Done is better than perfect. 完美的产品没人用，好用的产品有人爱
- **健康第一**：一人公司最大的风险是你倒下了。规律作息，别熬通宵
- **保持连接**：定期和真实用户交流，不要活在自己的 echo chamber 里
- **留后路**：关键数据和配置要有备份，关键决策要有 Plan B
