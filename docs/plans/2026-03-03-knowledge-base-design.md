# 知识库/素材库/案例库设计方案

**日期**: 2026-03-03  
**状态**: 已批准  
**作者**: AI Assistant  

## 1. 概述

### 1.1 目标
为标书生成系统增加知识库功能，让系统能够根据历史标书、企业资料和案例片段生成更贴合公司业务的专业标书内容。

### 1.2 核心需求
- **内容类型**: 历史标书文件、企业资料、案例片段
- **使用方式**: 智能推荐 - 系统自动检索并展示相关知识库条目，用户可以确认、取消或补充选择
- **内容管理**: 支持文件批量导入和手动单个编辑
- **检索技术**: 使用 PageIndex 进行语义推理检索
- **数据隔离**: 混合模式 - 有公共知识库（全局），也有企业/用户私有知识库

## 2. 技术方案选型

### 2.1 选定的方案：PageIndex

**选择理由**:
1. **与现有系统高度契合**: 项目已经基于 outline（目录结构），PageIndex 的树状索引与 outline 结构天然匹配
2. **更适合专业文档**: 标书是专业文档，需要领域知识和多步推理，PageIndex 的推理检索比向量相似度更准确
3. **更好的可解释性**: 用户可以理解为什么系统推荐这些知识库内容
4. **实施成本更低**: 无需引入向量数据库，无需嵌入 API 费用

**PageIndex 特点**:
- 无需向量数据库
- 无需分块处理
- 基于推理的检索
- 在 FinanceBench 上达到 98.7% 准确率

### 2.2 文件处理策略

- **PDF 文件**: 直接传给 PageIndex 处理
- **DOCX 文件**: 先转换为 PDF（使用 `docx2pdf` 或 `libreoffice`），然后调用 PageIndex
- **手动内容**: 转换为临时 Markdown 文件，然后用 PageIndex 处理

### 3.2 检索流程

```
用户点击"生成内容"
       ↓
1. 分析章节信息
   - 章节标题、描述、上级章节上下文
       ↓
2. 调用知识库检索服务
   - 遍历所有相关知识库条目
   - 使用 PageIndex 树搜索
   - LLM 推理判断相关性
       ↓
3. 展示推荐结果
   - 显示匹配的知识库条目
   - 用户可以确认/取消/补充选择
       ↓
4. 生成最终内容
   - 将选中的知识库内容注入提示词
   - 调用 LLM 生成章节内容
```

## 4. 数据模型设计

### 4.1 知识库条目表 (knowledge_items)

```python
class KnowledgeItem:
    id: UUID
    title: str                    # 标题
    type: str                     # 类型：history_bid / company_info / case_fragment
    scope: str                    # 范围：global / enterprise / user
    owner_id: UUID | None         # 所有者ID（企业或用户）
    
    # 内容
    content_source: str           # 来源：file / manual
    content: str | None           # 手动输入的内容
    file_path: str | None         # 原始文件路径
    file_type: str | None         # 文件类型：pdf / docx
    
    # PageIndex 索引
    pageindex_tree: JSON          # PageIndex 生成的树状索引
    pageindex_status: str         # pending / indexing / completed / failed
    pageindex_error: str | None   # 索引失败的错误信息
    
    # 元数据
    tags: List[str]               # 标签
    keywords: List[str]           # 关键词
    category: str | None          # 分类
    
    # 统计
    usage_count: int              # 使用次数
    last_used_at: datetime        # 最后使用时间
    
    # 时间戳
    created_at: datetime
    updated_at: datetime
```

### 4.2 知识库使用记录表 (project_knowledge_usage)

```python
class ProjectKnowledgeUsage:

## 5. 核心服务设计

### 5.1 PageIndex 集成服务 (PageIndexService)

**主要功能**:
- `process_pdf(pdf_path)`: 处理 PDF 文件，生成 PageIndex 树
- `process_markdown(md_path)`: 处理 Markdown 文件，生成 PageIndex 树
- `convert_docx_to_pdf(docx_path)`: 将 DOCX 转换为 PDF

**实现方式**:
- 通过子进程调用 PageIndex 命令行工具
- 支持异步处理，避免阻塞
- 错误处理和重试机制

### 5.2 知识库检索服务 (KnowledgeService)

**主要功能**:
- `search_relevant_knowledge()`: 检索相关知识库内容
- `_search_in_pageindex_tree()`: 在 PageIndex 树中进行推理搜索
- `create_knowledge_item()`: 创建知识库条目并生成索引

**检索算法**:
1. 获取用户可访问的知识库条目（全局、企业、个人）
2. 对每个条目使用 LLM 在 PageIndex 树中进行推理搜索
3. 计算相关性得分（0-1）
4. 过滤低相关性结果（< 0.3）
5. 按相关性排序，返回前 K 个结果

### 5.3 集成到内容生成流程

**修改点**:
- 修改 `OpenAIService._generate_chapter_content()` 方法
- 新增 `knowledge_context` 参数
- 将知识库内容注入到提示词中

**提示词模板**:

```

## 9. 风险和缓解措施

### 9.1 技术风险

**风险 1**: PageIndex 处理大文件可能耗时较长
- **缓解措施**: 
  - 异步处理，避免阻塞
  - 添加处理进度提示
  - 支持后台索引，用户可以稍后查看结果

**风险 2**: DOCX 转 PDF 可能失败
- **缓解措施**: 
  - 提供多种转换方案（docx2pdf、libreoffice）
  - 失败时给出明确的错误提示
  - 建议用户直接上传 PDF

**风险 3**: LLM 推理检索成本较高
- **缓解措施**: 
  - 使用缓存机制
  - 批量处理减少 API 调用
  - 可选择性地启用知识库检索

### 9.2 业务风险

**风险 1**: 用户可能不愿意花时间管理知识库
- **缓解措施**: 
  - 提供批量导入功能
  - 支持从历史标书自动提取
  - 简化手动输入流程

**风险 2**: 推荐内容可能不准确
- **缓解措施**: 
  - 用户可以手动调整推荐结果
  - 提供相关性得分和理由
  - 持续优化检索算法

## 10. 后续优化方向

1. **自动提取**: 从历史标书中自动提取优质段落入库
2. **智能分类**: 使用 LLM 自动为知识库内容打标签和分类
3. **使用分析**: 统计知识库使用情况，优化推荐算法
4. **版本控制**: 支持知识库内容的版本管理
5. **团队协作**: 支持多人协作编辑知识库

## 11. 总结

本设计方案基于 PageIndex 的推理检索技术，为标书生成系统提供了完整的知识库解决方案。方案的核心优势在于：

1. **精准检索**: 基于推理而非相似度，检索结果更相关
2. **易于使用**: 智能推荐，用户可以快速确认
3. **灵活管理**: 支持文件上传和手动输入两种方式
4. **可扩展性**: 支持全局、企业、个人三级数据隔离

实施周期预计 7-11 天，分四个阶段逐步推进。技术风险可控，业务价值明显。
